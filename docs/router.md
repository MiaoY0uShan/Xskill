# Xskill proactive router

Xskill is designed to activate automatically for non-trivial coding tasks.

The user does not need to say `Xskill`.

Manual override is still supported:

```text
Xskill: <task or idea>
```

## Routing

```text
clear small task
→ optimize-path
→ compiled execution brief
```

```text
vague idea
→ idea cards
→ user chooses
→ normal workflow
```

```text
large or multi-module task
→ question-requirements
→ delete-scope
→ semantic-architecture
→ optimize-path
→ shorten-iteration
→ compiled execution brief
```

```text
failed or blocked task
→ evidence-ledger
→ failure-to-smaller-task protocol
→ shorten-iteration
→ smaller compiled execution brief
```

```text
completed task
→ evidence-ledger
→ metrics
→ adaptive-improvement
→ schema-memory
```

## Rule

Use Xskill when scope, context, verification, or evidence matters.
